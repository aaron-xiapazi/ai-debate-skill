# AI辩论Skill测试报告

## 测试概况
- 共跑了10轮测试（Round 1 - Round 10）
- 话题：催生政策到底有没有用
- 所有结果保存在 `results/` 目录

## 最终模型配置

| 角色 | 文案里的名字 | 实际调用模型 | 为什么 |
|------|-------------|-------------|--------|
| 豆包 | 豆包 | deepseek-v3.2 | 中文数据最强，测试key无豆包文本模型 |
| GPT | GPT | gpt-4o | 直接可用 |
| Claude | Claude | claude-opus-4-6 | 直接可用 |
| Gemini | Gemini | deepseek-r1 | 犀利度最高，测试key无Gemini模型 |

## 模型表现 vs 文案预期

### 豆包 ✅ 完全匹配
- 文案说"哐哐甩数据，结论是需综合考量" → 模型输出大量省份数据、补贴金额，确实倾向于不敢下判断
- 互怼环节用数据反驳其他AI效果很好

### GPT ✅ 完全匹配
- 文案说"全程温柔共情，和稀泥" → 输出充满"理解年轻人""每个人都有选择"
- 互怼时居然也很犀利（意外惊喜）

### Claude ✅ 基本匹配
- 文案说"拆成三大块分析，像读论文" → 确实会结构化分析
- 问题：偶尔还是会用markdown格式（列表/加粗）
- 问题：有时开头会说"我是Claude"
- 互怼环节是四个里质量最高最稳定的

### Gemini ✅ 匹配（用deepseek-r1后）
- 文案说"上来就甩狠话" → "催生政策？那是用创可贴治疗癌症"
- 文案说"直接掀桌子" → "你们仨都在讨论怎么催，有没有人先问该不该催？"
- Round 1-3用qwen-max太温和，Round 4换deepseek-r1后效果炸裂
- 需注意：deepseek-r1偶尔会说过于敏感的话，已通过prompt限制

## 已知问题

1. **Gemini/豆包模型替代**：测试key无法调用真正的Gemini和豆包文本模型。如果有能调通的key，替换debate.mjs里MODELS配置的id即可
2. **Claude偶尔自我介绍**：虽然prompt禁止了，但claude-opus-4-6有时还是会说"我是Claude"
3. **融合字数不稳定**：目标600字左右，实际产出800-1200字之间波动。已将max_tokens限制到1200
4. **数据真实性**：AI（尤其是DeepSeek扮演的豆包）会编造具体数据。视频录屏时需注意标注"AI生成内容"

## 使用方法

```bash
cd skill-api/debate-skill
node debate.mjs "你的问题" "标签"
```

结果保存在 `results/debate_时间戳.md`
